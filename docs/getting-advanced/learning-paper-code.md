# How to learn the code of a paper or algorithm

In one sentence: read the code with a task in mind.

> When does reading code become very inefficient? When you read it without a goal.<br>In that case, even after you finish reading and you understand the logic, you still do not know which of your own needs the reading addressed, so you walk away with nothing.

Steps for learning the code of a paper or algorithm:

1. First be clear about why you are reading this code. Two common reasons:
	1. You have read the paper but you are not sure how a particular module actually works, so you want to read the code to find out.
	2. You want to borrow the algorithm from the paper and implement it yourself, so that you can use it to solve a problem in your own project.
2. Once the goal is clear, you usually have a clear direction when reading the code. You know which parts of the code to read, and you know what you are reading them for.

A trick that makes reading code more efficient (**works very well for beginners**):

1. Set yourself a small task that improves on the paper's algorithm, and then implement it.
2. The task pushes you to read the relevant code carefully, because you only know how to improve the code once you understand it.
3. While you implement the improvement, your own coding skills also get better.
