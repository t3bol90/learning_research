# How to Write Experiment Notes

> Documentation index (GitHub repo): [https://github.com/pengsida/learning_research](https://github.com/pengsida/learning_research)

An experiment note is usually organised like this. The wording does not have to be detailed, only clear enough that you can follow it later. Notes are written for yourself, after all.

1. **Goal of the experiment**: describe why you are running it and what you hope to learn.
2. **Experimental setting**: which dataset you ran it on, and what you changed in the algorithm.
3. **Record the results**: log both the results that worked and those that did not, including visualisations and quantitative numbers.
4. **Analyse the results**: check whether the outcome matches your expectations. If it does not, [work out why the experiment is not working](./why-experiment-isnt-working.md) (Notion).
5. **Next step**: you are the project lead. Keep thinking about what to do next, and list the experiments you plan to run rather than waiting for instructions.
